Display: Inline, Inline-Block or Block?

The CSS display property has several different values, which all interact in a different way with the contents of an element. Before we start, let's look at how we got here.

We start out with an element in HTML - for this example, I'll use a <span> (entirely because that's close to spaniel, and I wanted to make that pun). Seeing as a <span>(iel) is a container for just a small amount of content, it is probably nested inside of some <p> tags. So here's my <span> within a paragraph, and naturally, I've given it a class of spaniel. Within CSS, I can now give this spaniel class a display property.

display: inline;

Once I set the display for .spaniel to inline, I set my text colour to white and the background to brown. I've added some padding (extra brown) and some margin (extra surrounding white space). Inline display will size itself to cover the element it is set to - in this case our span. However, we can't set a fixed width or height for inline display - well, you could try, but it would be an exercise in futily, so I wouldn't bother :)

Funnily enough, inline display prioritises keeping the content of our span in line with the rest of the text. Therefore, the padding I added is overlapping the above content and sits behind content below (due to stack order), and my margin only functions horizontallly.


So much blog, so little dog... here's a little fuzzball to keep you going! Don't worry, this one's educational - that's the exticnt English Water Spaniel for reference :)


display: block;

As you might have guessed, display: block; is the opposite of display: inline; - instead of keeping the content of the span in line with the rest of the text, block sets it on a new line - it gives the content it's own block! Margin and padding are observed in both dimensions, and we can now set a fixed height and width. Note that the padding and margin are added to the edge of my specified height and width, not around the text.

Because I have a given the span a fixed height and width, the contents now overflow. For my example, this is inconvenient, but if you're using this display property on a larger piece of content, having set dimensions can be useful for how it displays - add an overflow: hidden; property and a scroll bar to manage the excess!

display: inline-block;

With display: inline-block; we get the best of both worlds (queue Hannah Montana please). The contents of my span stays in line with the rest of my paragraph, but I get to set width and height, and my margins and padding function in all directions.

We can see that the content of the span tries to fit within my given dimensions, but still overflows, which can be dealt with as explained above. Also note that though the span is in line with the rest of the paragraph, the margins I set are pushing the lines of the paragraph apart.
