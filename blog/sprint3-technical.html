JavaScript and the DOM

HTML and CSS

If we were to view the relationship between HTML and CSS as a tree, HTML would be the parts that make up the tree, in their most basic form. A trunk, some branches, roots, leaves and flowers. CSS arranges these parts and alters their appearance. It adds colour to the flowers so bees can find them easier, arranges the leaves for maximum sunlight exposure, determines the patterns in the bark. HTML gives us the bits and pieces, and CSS gives us the ability to style them, both for practicality and aesthetic.

The DOM

DOM stands for Document Object Model, which represents each component of the HTML document as an object so that JavaScript, an object-oriented language, can interact with it. Luckily for me, the DOM is pretty much a tree diagram, and thus my thematic consistency for this week is preserved.

In reality, the DOM is crazy complex and contains an object (called a node) for every tiny piece of information in the HTML document. It contains more information than we could every really need, but this simplified version gives us an idea of what it looks like.

Every element is a child, grandchild, great-grandchild... etc. of the document object. An element can have any number of children, but only ever has one parent (finally, the asexual representation we all needed.)

One way for JavaScript to access an element in the DOM is through IDs. In my HTML file, I could give an img element an id of tree, and then access it in JavaScript with document.getElementById('tree'). From there I can manipulate or make my image interactive.

Arrays vs. Objects

Both arrays and objects are types of special variables that contain collections of information, rather than just one piece of data. However, arrays are an (ordered) list of data points stored in a single variable, while an object should representation a 'thing' that has various characterisitics (known as properties).

On the left is an array, which contains a list of information inside the single variable treeBreeds.

On the right is an object, which contains information about the many properties of myTree.

Note that I can have an object within my array, and an array within my object (and on and on...)

Data in arrays is accessed with zero-indexing, which means that each piece of data in the array is numbered, starting from 0. As shown in the image, treeBreeds[1] correlates to 'Blue Gum'.

Meanwhile, objects are accessed by calling the key for the property we want the information from, which can be done using either dot notation or bracket notation. Note that the dot notation was simply myTree.heightInMeters, but when using bracket notation you must put the property name in quote marks, i.e. myTree['breed'].

We can put this together to call the information from within the nested objects/arrays. Returning treeBreeds[2].Magnolia.talents will return 'pretty flowers' from my array.

Functions

Functions are used for code we wish to repeat. We create the code for a specific task, assign that code to a function name (adding optional parameters), and then we call that function name (with arguments if necessary) whenever we want to use that set of code. This saves us from repeating code unnecessarily.

This function will test if someone isATreehugger and return a statement about them when passed the person's Name and if they careAboutClimateChange. Therefore, calling isATreehugger(true, Laché) would return "Yes, Laché is a treehugger, and should be proud of it."
